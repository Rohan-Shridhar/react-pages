  ## 🛠️ How To Begin Contributing

 For anyone new to contributing, here is a quick guide:
1. Comment on an issue
```text
    Hi! I'd like to work on this issue, please assign it to me.
````
 2. Fork this repository
- You can fork the repository by clicking "Fork" on the [main page](https://github.com/Rohan-Shridhar/react-pages) of the repository
3. Clone your fork
- Use the command
````bash
git clone https://github.com/YOUR-USERNAME/page-construction
````
4. Create a new branch
- Use the command
```bash
git checkout -b YOUR-BRANCH-NAME
````
5. Make your changes to the project
6. Stage, commit and push
- Use the command `git add file-name` to stage files you have changed
- Use the command `git commit -m "Insert a meaningful comment about your commit"` to commit
- Use the command `git push origin YOUR-BRANCH-NAME` to push

7. Open a Pull Request to send in your contributions
- Head to your fork on GitHub and open the Pull Requests tab to make a PR
- Make sure to follow the [PR template](https://github.com/Rohan-Shridhar/react-pages/blob/main/PULL_REQUEST_TEMPLATE.md)
        
## 🌿 Branch Naming Conventions

To keep the repository organized, please follow these naming rules when creating your branch:

| Type | Format | Example |
|------|--------|---------|
| New feature | `feature/short-description` | `feature/add-dark-mode` |
| Bug fix | `fix/short-description` | `fix/navbar-overflow` |
| Documentation | `docs/short-description` | `docs/update-contributing` |
| Chore / misc | `chore/short-description` | `chore/cleanup-unused-files` |

**Rules:**
- Use **lowercase letters** only
- Use **hyphens** (`-`) to separate words, not underscores or spaces
- Keep it **short but descriptive**
- Avoid generic names like `my-branch` or `patch-1`
